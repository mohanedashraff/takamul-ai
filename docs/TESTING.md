## ✅ أنواع الاختبارات

### 1. اختبار يدوي (بعد كل خطوة)
- افتح المتصفح
- جرب كل عنصر
- تأكد من RTL
- تأكد من Responsive (موبايل + تابلت + ديسكتوب)
- تأكد من عدم وجود أخطاء في Console
- تأكد من حالات التحميل (Loading states)
- تأكد من حالات الخطأ (Error states)

### 2. اختبار بالمتصفح (Browser Testing)
- استخدم webapp-testing skill
- التقط screenshots لكل صفحة
- تأكد من عمل الأنيميشنات
- تأكد من عمل النماذج (Forms)

### 3. اختبار API
```bash
# مثال: اختبار كاتب المقالات
curl -X POST http://localhost:3000/api/tools/content \
  -H "Content-Type: application/json" \
  -d '{"toolId": "article-writer", "topic": "الذكاء الاصطناعي", "length": 500}'
```

## 🔍 كيف تختبر

### الطريقة السريعة (كل خطوة):
1. `npm run dev`
2. افتح المتصفح على `localhost:3000`
3. جرب الـ feature الجديد
4. تأكد من عدم أخطاء في F12 Console
5. جرب على موبايل (F12 → Device Toolbar)

### الاختبار الشامل (نهاية كل مرحلة):
اقرأ القائمة المناسبة أعلاه وتأكد من كل نقطة.
